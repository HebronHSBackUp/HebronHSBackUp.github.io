Linear Collection: Queue and Priority Queue

Queues

Like stacks, queues are linear collections. However, insertions in a queue are restricted to one end, called the rear, and removals are restricted to the other end, called the front. A queue thus supports a first-in, first-out protocol (FIFO). Queues occur in everyday life from checkout lines in stores, to highway tollboth lines, and airport baggage check-in lines. A queue is used whenever people or things are lined up for processing on a first-come, first-serve basis.

Queues have two fundamental opertions: add, which adds an item to the rear of a queue, and remove, which removes an item from the front. (The traditional names of these operations are enqueue and dequeue)

Priority Queues

Related to queues is an ADT (Abstract Data Type) called a priority queue. A priority queue is essentially a list of items in which each item has associated with it a priority. In general, different items may have different priorities and we speak of one item having a higher priority than another. Given such a list we can determine which is the highest (or the lowest) priority item in the list. Items are inserted into a priority queue in any, arbitrary order. However, items are withdrawn from a priority queue in order of their priorities starting with the highest priority item first.

Most examples of queues in computer science involve scheduling access to shared resources, for instance,

  • CPU access - Processes are queued for access to a shared CPU.
  • Disk access - Processes are queued for access to shared secondary storage device.
  • Printer access - Print jobs are queued for access to a shared printer.

Interfaces and Implementing Classes

interface java.util.Queue<E>

boolean add(E obj) enqueues obj at the end of the queue; returns true
E remove() dequeues and returns the element at the front of the queue
E peek() returns the element at the front of the queue; null if the queue is empty
boolean isEmpty()  

class java.util.LinkedList <E> implements java.util.List<E>, java.util.Queue<E>

Note: use LinkedList class to create a queue

 

class java.util.PriorityQueue<E>

boolean add(E obj) adds obj to the priority queue; returns true
E remove() removes and returns the minimal element from the priority queue
E peek() returns the minimal element from the priority queue
boolean isEmpty()  

Note: E should implement Comparable interface

Using a Queue

To create a queue use the LinkedList class which implements the Queue interface.

Creating a Linked List Object

LinkedList<Integer> queue = new LinkedList();  // create queue

or

List<Integer> queue = new LinkedList(); // create queue

Adding an Object to a Queue

 
queue.add(57);  // adds item to rear of queue

Removing an Object from a Queue

int num = queue.remove();  // removes item from front of queue

Viewing the Top of a Queue

 
int num = queue.peek();  // does not remove the item

Traversing a Queue

while(!queue.isEmpty())
{
   System.out.println(queue.remove());
}

 

Using a Priority Queue

To create a Priority Queue use the Priority Queue class.

Creating a Linked List Object

PriorityQueue<Integer> queue = new PriorityQueue();  // create priority queue

Adding an Object to a Queue

queue.add(57);  // adds item to queue

Removing an Object from a Queue

int num = queue.remove();  // removes minimal value from queue

Viewing the Top of a Queue

int num = queue.peek();  // returns minimal value from queue

Traversing a Queue

while(!queue.isEmpty())
{
   System.out.println(queue.remove());
}
