Linear Collection: Stack

Stacks are linear collections in which access is completely restricted to just one end, called the top. The classical example is the stack of clean trays found in every cafeteria. Whenever a tray is needed, it is removed from the top of the stack, and whenever clean ones come back from the scullery, they are again placed on the top. No one ever takes some particular fine tray from the middle of the stack, and it is even possible that trays near the bottom are never used. Stacks are said to adhere to a last-in, first-out protocol (LIFO). The last tray brought back from the scullery is the first one taken by a customer.

JCF - Java Collections Framework

The java.util package includes a Stack class. The methods used to manipulate a stack are shown below. The operations for putting items on and removing items from a stack are called push and pop, respectively.

class java.util.Stack<E>

E push(E item) pushes item onto the top of the stack; returns item
E pop() removes and returns the element at the top of the stack
E peek() returns the element at the top of the stack
boolean isEmpty() throws an exception if the stack is empty

 

* Note: The Stack class provided by the JCF is a subclass of another class called Vector (similar to an ArrayList). An unfortunate side effect of this implementation is that clients can treat stacks as vectors by inserting, replacing, or removing an element at any position, thus violating the spirit of a stack which only allows manipulation from the top.

Using a Stack

Creating a Linked List Object

Stack<Integer> myStack = new Stack();

or

List<Integer> myStack = new Stack();

Adding an Object to a Stack

myStack.push(57);

Removing an Object from a Stack

int num = myStack.pop();

Viewing the Top of a Stack

int num = myStack.peek();  // does not remove the item

Traversing a Stack

while(!myStack.isEmpty())
{
   System.out.println(myStack.pop());
}

 

Applications of Stacks

Applications of stacks in computer science are numerous. Following are just a few examples:

  • Parsing expressions in context-free programming languages - a problem in compiler design
  • Translating infix expressions to postfix form and evaluating postfix expressions
  • Backtracking algorithms
  • Managing computer memory in support of method calls
  • Supporting the "undo" feature in text editors, word processors, spreadsheet programs, etc.
  • Maintaining a history of the links visited by a Web browswer

 

Analyzing the Performance of Stack Implementations

The run-time performance of a stack implementation depends on the run-time performance of the operations on the data structure contained in it. Stacks are linear collections, so the natural choices for implementation are based on arrays (ArrayLists) and linked structures (LinkedList).

A stack that uses an array list or an array can push or pop elements in constant time O(1), with one exception. When the underlying array runs out of room, it must be resized, and this operation takes linear time.

On the other hand, elements can always be added to the end or the beginning of a linked structure in constant time. A linked structure requires three times the memory to represent the links and references to the data but no extra memory to represent the data themselves. Thus, where time is of the essence, a linked implementation is usually preferable.
